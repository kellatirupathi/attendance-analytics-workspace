# BUILD: NIAT SPI вЂ” Skill Performance Index Platform (Replit)

Build a full-stack platform that reproduces an existing Next.js app **feature-for-feature, layout-for-layout, style-for-style**, but on a new stack. Attendance/quiz/skill data is read **LIVE from Google BigQuery**. PostgreSQL stores only users, roles, campuses and scoping. **There is NO MongoDB and NO background sync job** вЂ” do not build any "fetch to DB every N minutes then read DB in the UI" pattern.

## 0. Tech stack (use EXACTLY this)
- **Monorepo:** pnpm workspaces. TypeScript 5.9. Node 24.
- **Backend:** `artifacts/api-server` вЂ” Express 5 REST API on **port 8080**, all routes mounted under **`/api`**. Bundled to CJS with **esbuild**. Auth via **Passport** (Replit OIDC + NIAT Forms SSO) **and** bcrypt admin/staff password login. **pino** for logging.
- **DB layer:** `lib/db` вЂ” **PostgreSQL** + **Drizzle ORM** + **drizzle-kit** (migrations) + **drizzle-zod** (schemaв†’Zod). Drizzle schema lives in **`lib/db/src/schema/brave.ts`**.
- **Frontend:** `artifacts/brave-dashboard` вЂ” **React + Vite**, **Tailwind CSS + shadcn/ui (Radix)**, **wouter** routing, **TanStack React Query** for fetching/caching, **Zod** for validation. Charts with **recharts**. Icons with **lucide-react**. Dates with **date-fns**.
- In production, the Express server also serves the built Vite frontend (static) and handles `/api/*`; in dev, Vite proxies `/api` в†’ `http://localhost:8080`.

## 3. PostgreSQL schema (`lib/db/src/schema/brave.ts`, Drizzle)

Generate drizzle-zod insert/select schemas for validation. Empty `campuses`/`subjects` arrays mean "unrestricted" (interpreted per role in RBAC).

## 5. BigQuery integration (LIVE вЂ” the only source for attendance/quiz data)
Port these helpers to `artifacts/api-server/src/lib/bigquery.ts`:

- **Auth:** build a signed RS256 JWT from `BIGQUERY_SERVICE_ACCOUNT_JSON` (client_email/private_key), exchange at `https://oauth2.googleapis.com/token` for an access token (scope `bigquery.readonly`), cache token in memory until ~30s before expiry.
- **`bqQuery(sql, params?, location?)`**: POST to `bigquery/v2/projects/{BQ_PROJECT_ID}/queries` with `useLegacySql:false`, `timeoutMs:30000`, `location: BQ_LOCATION`. **Always use NAMED parameters** вЂ” never string-concat user input. Param typing: `numberв†’INT64`, `stringв†’STRING`, `string[]в†’ARRAY<STRING>` and reference via `IN UNNEST(@name)`.
- Metadata helpers (superadmin BigQuery Explorer): `listDatasets`, `listTables(dataset)`, `getTablePreview(dataset, table, limitв‰¤200)`. For the preview of views, **validate `dataset`/`table` against an allowlist from `listTables` or `^[A-Za-z0-9_]+$`** before interpolating into SQL (avoid identifier injection). Return generic errors to the client (don't leak `error.message`).
- Input helpers: `validateStudentId = /^[a-zA-Z0-9_\-]{4,64}$/`, `pct(present,total)=round to 0.1`.

**Tables:**
- Attendance: `` `kossip-helpers.niat_post_onboarding_engagement_ai_analytics_workspace.z_niat_student_session_wise_attendance_details` ``
- **Quizzes (NEW):** `` `kossip-helpers.niat_post_onboarding_engagement_ai_analytics_workspace.z_niat_students_classroom_and_module_quiz_details` ``

**Attendance queries** (`src/lib/queries.ts`, all filter `is_current_semester = 1`, scoped via `scopeClause`):

- `getStudentSubjects(studentId)` вЂ” per subject_title present/total/%, meetsRequirement (в‰Ґ80).
- `getStudentRecentSessions(studentId)` вЂ” up to 500 rows, `date, session_title, subject_title, attendance_status, marking_method`, newest first.
- `searchStudents(q, limitв‰¤50, scope)` вЂ” name LIKE or exact id.
- `getStudentsList(scope, {search,limitв‰¤1000})` вЂ” grouped per student, ordered by attendance % asc.
- `getCampusSummary / getSubjectSummary / getSectionSummary(scope)` вЂ” rollups (students, sections, subjects, present/total, %).

**Quiz queries (NEW):**

## 6. RBAC (`src/lib/rbac.ts`)
- `canManage(role)` в†’ only `superadmin | admin`.
- `manageableRoles(role)`: superadmin в†’ [admin,hod,capability_manager,boa,instructor]; admin в†’ [hod,capability_manager,boa,instructor]; others в†’ [].
- `scopeForSession(session)`: superadmin/admin/hod в†’ `{}` (unrestricted); boa в†’ `{campuses}`; capability_manager & instructor в†’ `{campuses, subjects}`; default в†’ `{campuses:['__none__']}`.
- Role labels & descriptions (reuse the exact strings from the reference: Super Admin, Admin, HOD, Capability Manager, BOA, Instructor). `REQUIRED_PCT = 80`.

## 7. Auth (`src/lib/auth.ts`, Passport, JWT cookie)
- **Session:** signed **HS256 JWT** (jsonwebtoken/jose) carrying `{sub,email,name,role,campuses,subjects}`, 30-day expiry, set as httpOnly cookie `niat_session` (`secure` in prod, `sameSite:lax`, `path:/`, maxAge 30d).
- **Password login** `POST /api/auth/login`: rate-limit 10/min/IP; lookup active user by lowercased email; `bcrypt.compare`; on success set cookie + update `lastLoginAt`; generic "Invalid email or password" on failure.
- **OIDC/SSO (Passport):** Google OIDC and NIAT Forms SSO / Replit OIDC strategies. Policy: **only sign in emails that already exist as `isActive` users вЂ” never auto-create.** OAuth `state` = short-lived signed JWT carrying validated `next` (must start with `/`, else `/dashboard`). Verify `email_verified`.
- `GET /api/auth/me` (session profile), `POST /api/auth/logout` (clear cookie).
- **Revocation (improve on the original):** add `tokenVersion` to users; embed it in the JWT and re-check it (or `isActive`) in `requireSession`, so deactivating/role-changing a user invalidates live sessions instead of waiting 30 days.

Express middleware `requireSession({manage?})` в†’ 401 if no/invalid session, 403 if `manage` and not `canManage`. Guard all `/dashboard` and `/admin` **pages** client-side (redirect to `/staff-login`) AND guard every `/api` route server-side (defense in depth).
