### A2. Backend query + endpoint
In the queries lib add `getStudentQuizzes(studentId)` using **named parameters** (never string-concat):
```
returns {
  classroom: { avgPercentage: number, attempted: number, total: number,
               items: [{ title, subjectTitle, score, maxScore, percentage, status, date }] },
  module:    { avgPercentage: number, attempted: number, total: number,
               items: [{ title, subjectTitle, score, maxScore, percentage, status, date }] }
}
```
- Filter `student_user_id = @studentId` (+ `is_current_semester = 1` if the column exists).
- Split into classroom vs module by the kind column.
- `percentage = maxScore > 0 ? round(score/maxScore*1000)/10 : 0`; `status = completed/attempted ? 'COMPLETED' : 'PENDING'`.
- `avgPercentage` = mean of item percentages (0 if none). `attempted` = count with a real score; `total` = item count.
- Wrap in the same in-memory TTL cache as attendance (key `quizzes:<studentId>`, ~5 min).

Route (public via SPI token OR staff session, exactly like the other attendance endpoints — reuse `allowSpiAccess` + `validateStudentId`):
```
GET /api/attendance/students/:studentId/quizzes   → getStudentQuizzes(studentId)
```
Return generic errors on failure; never leak `error.message`.
